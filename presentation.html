02 · Technology Stack

Languages, core frameworks, database engines, and design methodologies

02 / 09
💻 Frontend System

Built with HTML5, JavaScript (ES6+), and Vanilla CSS3. Utilizes a modular design token architecture, responsive variables, custom blur filters, glassmorphic grids, and CSS transition animations.

☕ Java MVC Backend

Driven by Java Servlets (Jakarta EE) handling URL routing and transactional logic, working with JSPs (JavaServer Pages) providing state-aware presentation rendering.

🗄️ Database Layer

Powered by MySQL. Employs a fully normalized relational schema, primary/foreign key connections, index paths, and handles queries via standard JDBC connection drivers.

⚙️ Server & Tools

Hosted locally using Apache Tomcat Server. Compiled, structured, and managed inside the Eclipse IDE environment, with version control managed via Git and GitHub.

03 · Project Structure

MVC folder organizational map showing the division of code roles

03 / 09
Project Structure Directory

📂 MVC Package Architecture

Strict division of responsibilities ensures maintainability. Standard Models map directly to MySQL tables, while DAO layers decouple the business code from JDBC statements.

📂 Web Assets separation

JSP files retrieve session profiles and execute server-side checks, leaving high-performance UI state manipulations (like Cart caching) to vanilla JavaScript.

08 · Major Challenge Faced & Resolution

How a critical architectural bottleneck was solved during backend integration

08 / 09
⚠️ The State Bridge Bottleneck

Problem: LocalStorage is fast but traditional Java Servlets expect form parameters and cannot parse nested client-side JSON maps without deploying massive third-party parser libraries.

✅ Form Array Serialization Solution

Resolution: Engineered a serialization layer. JavaScript loops through cart items, dynamically appends parallel arrays as hidden DOM fields, and submits natively for servlet mapping.

Challenge: Cart State Caching
